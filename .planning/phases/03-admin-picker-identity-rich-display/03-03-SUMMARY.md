## Implementation Decisions

### onMouseDown + preventDefault vs blur-delay — chose both

Per UI-SPEC §"Keyboard Interaction Model": dropdown rows use `onMouseDown={e => { e.preventDefault(); add(u.email); }}`. The `preventDefault()` lets the row commit BEFORE the input's `onBlur` fires, avoiding the "click swallowed by blur-close" footgun.

The 100ms blur delay (`setTimeout(() => setOpen(false), 100)`) is a belt-and-braces — `onMouseDown` already handles the mouse path, but the delay covers any activation that arrives via touch or keyboard (where `onMouseDown` doesn't fire before blur).

**Choice rationale:** Either approach alone is enough on desktop, but using both makes the picker resilient on touch devices and under React 18 strict-mode double-effects (the chosen pattern is idempotent — calling `add` once and `setOpen(false)` once produces the same result regardless of order).

### vitest.config.ts include glob — permanent change

Plan 04 and Plan 05 will likely co-locate component-level tests too. The glob `components/**/*.test.ts` is now part of vitest.config.ts — future component test files automatically run as part of the suite without further config changes.

### @vitejs/plugin-react devDependency — required, not optional

Next.js's `tsconfig.json` sets `jsx: "preserve"` (the standard for App Router). When `vitest` loads `components/ui/PrimeUserPicker.tsx` via `vite-tsconfig-paths`, the bundled esbuild loader honors that setting and refuses to parse JSX as anything other than literal text → `Failed to parse source for import analysis`.

Considered alternatives:
1. **Move helpers to a separate `.ts` file and re-export from `.tsx`.** Rejected — the plan's `must_haves.artifacts` pins `filterPrimeUsers` + `normalizeManualEmail` as direct exports of `PrimeUserPicker.tsx`. Acceptance criteria require the literal `export function filterPrimeUsers(` pattern in that file. Re-exports would violate the contract.
2. **Override `esbuild.jsx: 'automatic'` in vitest.config.ts.** Tried — failed because `vite-tsconfig-paths` reads tsconfig and continues to honor `jsx: preserve`.
3. **Install `@vitejs/plugin-react`.** Chosen — adds one dev dependency, smallest possible change, decouples test JSX handling from production bundler config.

The plugin is dev-only (devDependency); production builds (Next.js `next build`) are unaffected.
